# AI-агент для сравнения городов по релокации

Проект помогает сравнить города для переезда или удаленной работы в формате обычного диалога.
Пользователь пишет свободный запрос на русском или английском, а приложение:

1. Определяет намерение (сравнение, климат, данные о стране или общий вопрос).
2. Собирает фактические данные из внешних API.
3. Показывает результат в таблице и дает рекомендацию по выбранному профилю.

## Что умеет приложение

1. Сравнивать два города по ключевым метрикам: страна, регион, валюта, часовой пояс, население, климат.
2. Отвечать на отдельные вопросы о стране/городе.
3. Выдавать климатические данные по городу.
4. Давать общие ответы через LLM, если запрос не относится к инструментам.

## Какие API используются

1. Open-Meteo Geocoding API
  https://geocoding-api.open-meteo.com/v1/search

  Для:
  - нормализации и уточнения города,
  - получения страны по городу,
  - получения таймзоны в виде IANA,
  - получения населения города

2. Open-Meteo Archive API
  https://archive-api.open-meteo.com/v1/archive

  Для:
  - среднемесячной температуры,
  - суммарных осадков за период с начала текущего месяца до текущей даты.

3. Open-Meteo Forecast API (fallback)
  https://api.open-meteo.com/v1/forecast

  Для:
  - резервного расчета климата, если архив недоступен по timeout/HTTP ошибке.

4. REST Countries API v3.1
  https://restcountries.com/v3.1

  Для:
  - названия страны,
  - валюты,
  - региона,
  - часовых поясов страны,
  - населения страны,
  - столицы.

5. Groq (LLM)
  https://api.groq.com/openai/v1

  Для:
  - классификации intent,
  - выделения города и страны из свободного запроса,
  - генерации общего ответа в режиме general.

Примечание: валютная конвертация через exchangerate-api из текущей версии удалена.

## Архитектура проекта

### 1) Слой оркестрации: main.py

В этом файле собран основной runtime приложения.

Ключевые обязанности:

1. Инициализация LLM из переменных окружения.
2. Читать запрос пользователя, обработать, отрисовать результат.
3. Intent routing:
  - сначала попытка через LLM,
  - затем fallback на эвристики.
4. Вызов инструментов из tools.py.
5. Финальный вывод и запись trace в agent_trace.jsonl.

### 2) Слой инструментов и интеграций: tools.py

В этом файле сосредоточены внешние запросы, кэш и сбор структурированных данных.

Ключевые обязанности:

1. Извлечение города/страны через LLM helper.
2. Геокодинг и определение страны/координат через Open-Meteo.
3. Сбор country data через REST Countries.
4. Сбор climate data через Open-Meteo archive/forecast.
5. Кэширование ответов в cache.json
6. Обработка ошибок и fallback.

### 3) Конфигурационный слой

1. .env / .env.example
2. requirements.txt
3. cache.json и agent_trace.jsonl

## Подробная логика обработки запроса

Ниже реальный pipeline в рабочей версии:

1. Пользователь вводит произвольный текст.

2. Определяется intent:
  - compare
  - country
  - climate
  - general
  - help
  - exit

3. Для compare:
  1) Вызывается get_country_data для каждого города.
  2) Вызывается get_climate_data для каждого города.
  3) Строится сравнительная таблица.
  4) Считается score и формируется рекомендация.

4. Для country:
  - один вызов get_country_data и вывод полей.

5. Для climate:
  - сначала определяется страна (если не указана),
  - затем вызывается get_climate_data.

6. Для general:
  - ответ от LLM без вызова инструментов данных.

7. Любой результат логируется в agent_trace.jsonl:
  timestamp, input, tool_calls, preview output.

## Как считается рекомендация

Сравнение не делается на глаз LLM, используется явная формула score.

Входные признаки:

1. avg_temperature
2. precipitation
3. currency (через бонус strong_score)
4. профиль пользователя (warm/cold/quiet/remote/neutral)

Логика профиля:

1. Если профиль содержит "удал", "remote", "фриланс", "digital", "номад" -> remote formula.
2. Если профиль содержит теплые/холодные/семейные ключевые слова -> соответствующая формула.
3. Если профиль не распознан -> нейтральный режим.

Бонус по валюте:

1. Для USD, EUR, GBP, CHF, CAD, AUD -> 2.0
2. Для остальных -> 0.5

Побеждает город с большим итоговым score.

## Запуск

```bash
pip install -r requirements.txt
python main.py
```

Примеры запросов:

1. Сравни Москву и Берлин для удаленки
2. compare madrid and lisbon for digital nomad
3. Какая валюта в Японии
4. Какой климат в Торонто
5. help
6. exit

## Переменные окружения

1. GROQ_API_KEY
2. LLM_MODEL (по умолчанию llama-3.3-70b-versatile)

Если ключ не задан, приложение продолжает работать на эвристиках и API данных, но ответы режима general будут ограничены.

## Ограничения текущей версии

1. Качество гео-резолва зависит от корректности названия города и доступности Open-Meteo.
2. При нестабильном интернете возможны timeout/ошибки API.
3. Для неоднозначных городов возможны пограничные кейсы (например, несколько городов с одним названием).
4. Рекомендация основана на простой прозрачной формуле, а не на сложной многокритериальной модели.
